# debug

## 目录定位
运行时调试工具目录。

## 职责范围
- 显式 runtime 调试辅助
- 开发联调用工具
- 非正式流程的诊断能力

## 允许放入
- debug only 脚本
- 开发态 runtime 诊断工具

## 禁止放入
- 正式流程主逻辑
- battle 规则真相
- 需要常驻线上链路的核心实现

## 对外依赖
- 可依赖 `res://app/flow/` 和已存在的正式 runtime
- 不应成为其它目录的正式依赖核心

## 维护约束
- 文件命名要体现 debug/runtime 语义
- 工具必须显式可控
- 不继续使用 phase 命名
